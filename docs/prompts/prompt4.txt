Epic 3 Goal:

Add a rule-based eligibility evaluation capability so that a submitted loan application can be evaluated using transparent demo banking rules.

This is a demo rule engine for portfolio and architecture purposes. Do not represent it as a real banking credit approval engine.

Eligibility rules for MVP:

Use these demo rules unless the existing implementation already defines a better documented rule set:

1. Minimum monthly income: 25,000
2. Maximum debt-to-income ratio: 50%
3. Maximum requested amount: 20 times declared monthly income
4. Minimum tenure: 6 months
5. Maximum tenure: 84 months
6. Existing EMI obligations cannot be greater than declared monthly income

Eligibility decision logic:

* If all mandatory rules pass, decision should be `Passed`.
* If one or more mandatory rules fail, decision should be `Failed`.
* The response must clearly explain which rules passed and which rules failed.
* Include calculated values such as debt-to-income ratio where useful.
* Include rule version, evaluated timestamp, and correlation ID.

Suggested eligibility result fields:

* Eligibility result ID
* Loan application ID
* Customer ID
* Decision: Passed or Failed
* Rule version
* Evaluated timestamp
* Requested amount
* Declared monthly income
* Existing EMI obligations
* Debt-to-income ratio
* Decision summary
* Rule results collection

Suggested rule result fields:

* Rule code
* Rule name
* Passed true/false
* Actual value
* Expected value or threshold
* Explanation

Suggested API endpoints:

* `POST /api/v1/eligibility/check`
* `GET /api/v1/eligibility/applications/{applicationId}`
* `GET /api/v1/eligibility/results/{eligibilityResultId}`

Use consistent API response and error response patterns according to `AGENTS.md`.
