Implement Epic 1: Customer Registration based on AGENTS.md and the existing repository foundation.

Build the first business feature for the `enterprise-loan-origination-platform`: Customer Registration.

Before implementation:

1. Read `AGENTS.md`.
2. Review the existing repository structure created in Epic 0.
3. Review existing architecture documents, README, tests, CI workflow, and Angular structure.
4. Preserve the architecture direction and quality standards already defined.

Epic 1 Goal:

Allow a new customer to register with basic profile and financial summary details through backend APIs and Angular frontend.

Backend scope:

1. Implement Customer Registration in `Customer.Api`.
2. Create a clean domain model for `Customer`.
3. Add request and response DTOs.
4. Add validation for all required fields.
5. Add persistence using Entity Framework Core.
6. Add database migration support.
7. Add synthetic seed data if appropriate.
8. Add Swagger/OpenAPI documentation readiness.
9. Add structured logging.
10. Add correlation ID support if already available from Epic 0.
11. Add health check readiness if not already present.
12. Add proper error handling using Problem Details or the existing error-handling pattern.
13. Add unit tests and API/integration tests where practical.

Customer registration fields:

* Full name
* Email
* Mobile number
* Date of birth
* Employment type
* Monthly income
* Existing monthly obligations

Validation rules:

* Full name is required.
* Email is required and must be valid.
* Mobile number is required and must be valid enough for demo use.
* Date of birth is required.
* Customer must be at least 18 years old.
* Employment type is required.
* Monthly income must be greater than zero.
* Existing monthly obligations cannot be negative.
* Existing monthly obligations should not be greater than monthly income.

Suggested API endpoints:

* `POST /api/v1/customers`
* `GET /api/v1/customers/{customerId}`
* `GET /api/v1/customers`

Use consistent API response and error response patterns according to `AGENTS.md`.

Frontend scope:

1. Implement Customer Registration UI in Angular.
2. Use a feature-based folder structure.
3. Use reactive forms.
4. Add form-level and field-level validation.
5. Show user-friendly validation messages.
6. Show loading state during submission.
7. Show success message after successful registration.
8. Show API error message when registration fails.
9. Add a customer list or customer lookup view if it fits naturally.
10. Ensure UI looks professional, clean, and suitable for a banking platform demo.
