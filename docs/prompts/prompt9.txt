Implement Epic 8: DevOps and Docker based on AGENTS.md and the existing repository.

Build the next engineering capability for the `enterprise-loan-origination-platform`: Dockerized Local Development, CI Validation, and DevOps Foundation.

Before implementation:

1. Read `AGENTS.md`.
2. Review the current repository structure.
3. Review the implementation from Epic 0 through Epic 7.
4. Review existing backend services, Angular frontend, tests, documentation, CI workflow, Docker files, observability setup, and Azure-readiness notes.
5. Preserve the existing architecture direction, naming conventions, API response pattern, security standards, testing standards, observability standards, and Azure deployment direction.

Epic 8 Goal:

Make the platform easy to build, test, run, and validate using Docker and GitHub Actions.

This epic should demonstrate enterprise delivery maturity, local developer experience, repeatable builds, CI quality gates, containerization readiness, and Azure Container Apps deployment readiness.

Do not deploy to Azure in this epic. Prepare the project for future Azure deployment.

Backend Docker scope:

1. Add or standardize Dockerfiles for all backend APIs and workers.
2. Use multi-stage Docker builds where practical.
3. Ensure backend services can run in containers.
4. Ensure containers use environment-based configuration.
5. Ensure services expose documented ports.
6. Ensure health check endpoints are usable inside containers.
7. Do not hardcode secrets.
8. Do not include development-only secrets inside images.
9. Optimize Dockerfiles for clean, repeatable builds.
10. Add `.dockerignore` files where appropriate.

Backend services to consider:

* Customer.Api
* LoanApplication.Api
* Eligibility.Api
* Notification.Worker or Notification service
* Audit.Api

If some services are not fully implemented yet, create a clean Docker-ready foundation and document any limitations.

Frontend Docker scope:

1. Add Dockerfile for Angular frontend.
2. Use production-style Angular build.
3. Serve the built Angular app using a lightweight web server such as Nginx if appropriate.
4. Keep environment configuration flexible for API base URLs.
5. Do not hardcode backend URLs unless they are local Docker defaults and documented.
6. Add `.dockerignore` for frontend if appropriate.

Docker Compose scope:

1. Update or create `docker-compose.yml`.
2. Add services for backend APIs, workers, frontend, and SQL Server.
3. Add local networking between services.
4. Add environment variables for local development.
